## 一、产品概述与目标用户

### 1.1 产品背景

在现代互联网、金融科技和物联网应用中，每天都会产生海量的时序指标数据（如日活跃用户数、API响应延迟、交易金额、传感器读数等）。这些数据蕴含着巨大的业务价值，但当前普遍存在以下问题：

- 数据散落在不同系统和数据库中，缺乏统一的查询入口；
- 数据分析停留在“查表看数”层面，缺乏自动化的异常检测和趋势洞察；
- 对未来走势的预测依赖人工经验，缺乏数据驱动的科学依据；
- 现有监控工具（如Grafana）侧重可视化而非分析，定制化统计能力弱。

### 1.2 产品定位

**智能指标分析平台** 是一个轻量级、API优先的时序指标分析引擎。它提供从数据接入、存储、统计分析到异常检测、趋势预测的完整能力，并以RESTful API的形式对外提供服务，方便嵌入现有业务系统。

### 1.3 目标用户

| 用户角色         | 描述                                 | 核心诉求                                          |
| :--------------- | :----------------------------------- | :------------------------------------------------ |
| **数据工程师**   | 负责数据ETL和系统集成的技术人员      | 通过API快速接入指标数据，获取分析结果用于下游系统 |
| **业务运维人员** | 负责监控业务健康状况的运营人员       | 及时发现异常指标，获取预警信息                    |
| **数据分析师**   | 对业务数据进行探索和分析的数据从业者 | 快速获取统计特征、发现数据规律、辅助决策          |
| **产品经理**     | 关注产品核心指标变化的产品负责人     | 了解趋势、预测未来，为产品规划提供数据支撑        |

### 1.4 产品愿景

> 让每一个指标数据都能被自动理解、分析和预警，将数据洞察能力嵌入到每一个业务系统中。

### 1.5 成功标准（OKR）

| 目标           | 衡量指标                       | 目标值  |
| :------------- | :----------------------------- | :------ |
| 数据接入便捷性 | 从上传到查询的完整链路打通时间 | < 5分钟 |
| 分析响应速度   | 统计查询接口P95响应时间        | < 100ms |
| 异常检测准确率 | 人工标注的异常样本召回率       | > 85%   |
| 系统可用性     | 核心API可用性                  | > 99.5% |

## 二、详细功能列表

### 模块一：指标数据管理

| 功能ID   | 功能名称       | 功能描述                                                     |
| :------- | :------------- | :----------------------------------------------------------- |
| **F1.1** | 创建指标       | 用户通过API创建一个新的指标定义，包含指标名称、单位、描述等元信息 |
| **F1.2** | 批量上传数据点 | 通过API批量上传某个指标的时序数据点（时间戳 + 数值）         |
| **F1.3** | 单点追加       | 在已有指标后追加一个新的数据点，用于实时数据流的接入         |
| **F1.4** | 查询原始数据   | 按指标名称和时间范围查询原始数据点，支持分页和排序           |
| **F1.5** | 查询最新值     | 获取某个指标的最新一个数据点，用于Dashboard展示              |
| **F1.6** | 删除指标       | 删除一个指标及其所有数据点（软删除，保留审计日志）           |
| **F1.7** | 指标列表       | 查询系统中所有已定义的指标及其元信息                         |

### 模块二：统计分析引擎

| 功能ID   | 功能名称      | 功能描述                                                     |
| :------- | :------------ | :----------------------------------------------------------- |
| **F2.1** | 基础统计      | 计算指标的均值、中位数、最大值、最小值、总和、标准差、数据量 |
| **F2.2** | 分位数计算    | 计算指定分位数（如P50、P75、P90、P95、P99）                  |
| **F2.3** | 移动平均      | 计算指定窗口大小的简单移动平均线，用于平滑趋势               |
| **F2.4** | 滑动窗口统计  | 对时间窗口内的数据进行滚动统计（如最近24小时的平均值）       |
| **F2.5** | 同比/环比计算 | 计算与上一周期（如昨天、上周同期）相比的变化率和变化量       |
| **F2.6** | 数据分布      | 返回数据的直方图分桶信息，用于了解数据分布形态               |

### 模块三：异常检测（🌟 核心亮点）

| 功能ID   | 功能名称    | 功能描述                                                     |
| :------- | :---------- | :----------------------------------------------------------- |
| **F3.1** | 3σ异常检测  | 基于拉依达准则，检测超出均值±3倍标准差的异常点               |
| **F3.2** | IQR异常检测 | 基于四分位距法，检测低于Q1-1.5IQR或高于Q3+1.5IQR的异常点（可选） |
| **F3.3** | 突变检测    | 检测数据序列中发生突变的点（前后差值超过设定阈值）           |
| **F3.4** | 异常汇总    | 返回指定时间段内的所有异常点列表及异常程度评分               |
| **F3.5** | 异常配置    | 允许用户自定义异常检测的阈值参数（如σ倍数、IQR倍数）         |

### 模块四：趋势预测（差异化亮点）

| 功能ID   | 功能名称       | 功能描述                                                     |
| :------- | :------------- | :----------------------------------------------------------- |
| **F4.1** | 提交预测任务   | 用户指定指标和预测步长，系统返回task_id并异步执行预测        |
| **F4.2** | 查询任务状态   | 通过task_id查询预测任务的执行进度和状态                      |
| **F4.3** | 获取预测结果   | 任务完成后，通过task_id获取预测结果（未来N个时间点的预测值及置信区间） |
| **F4.4** | 线性回归预测   | 基于历史数据进行线性趋势外推预测（快速、轻量）               |
| **F4.5** | 季节性分解预测 | 使用STL或简单季节性模型进行预测（识别周期性模式）            |
| **F4.6** | 预测任务列表   | 查询用户提交的所有预测任务的历史记录                         |

### 模块五：对比分析

| 功能ID   | 功能名称     | 功能描述                                                   |
| :------- | :----------- | :--------------------------------------------------------- |
| **F5.1** | 相关性分析   | 计算两个指标之间的皮尔逊相关系数                           |
| **F5.2** | 指标叠加查询 | 同时查询多个指标在相同时间段的数据，用于对比展示           |
| **F5.3** | 汇总对比     | 对比多个指标的统计值（均值、最大值、趋势等），输出对比表格 |

## 四、界面设计要求

### 4.1 设计原则

由于本产品是 **API优先** 的后端服务，界面设计主要围绕 **API文档** 和 **管理控制台**（可选）展开：

| 原则       | 说明                                                         |
| :--------- | :----------------------------------------------------------- |
| **清晰性** | API文档必须清晰展示每个端点的路径、参数、请求体、响应格式和错误码 |
| **可交互** | 提供Swagger UI或Postman Collection，支持在线调试             |
| **简洁性** | 如有管理控制台，保持极简风格，聚焦于指标管理和任务监控       |

### 4.2 API文档界面要求

- 使用FastAPI内置的 **Swagger UI** (`/docs`) 和 **ReDoc** (`/redoc`)；
- 所有接口必须有完整的 **描述信息**（summary 和 description）；
- 请求体和响应体必须使用 **Pydantic Schema** 定义，并添加字段描述；
- 所有错误码必须在文档中统一列出和说明。

### 4.3 管理控制台界面（可选，v2.0）

如果后续开发简单Web管理界面，需满足：

| 页面           | 内容                                                         |
| :------------- | :----------------------------------------------------------- |
| **Dashboard**  | 展示系统总览：指标总数、数据点总数、近期异常数量、任务队列状态 |
| **指标列表页** | 表格展示所有指标，支持按名称搜索，点击进入详情               |
| **指标详情页** | 展示指标元信息、数据预览表格、触发异常检测和预测任务的按钮   |
| **任务监控页** | 展示所有预测任务的状态列表（进行中/已完成/失败）             |

## 六、非功能性需求

### 6.1 性能需求

| 指标             | 要求                     | 说明                                                         |
| :--------------- | :----------------------- | :----------------------------------------------------------- |
| **API响应时间**  | P95 < 100ms              | 统计查询接口（含缓存命中）；不含缓存的复杂计算可放宽至P95 < 500ms |
| **吞吐量**       | 单实例 > 500 QPS         | 适用于简单读写接口；复杂分析接口可接受更低QPS                |
| **数据上传性能** | 单次支持 10,000 条数据点 | 批量上传接口的性能目标                                       |
| **预测任务耗时** | 单指标预测 < 30秒        | 对于1000个数据点的指标                                       |
| **并发连接数**   | 支持 > 100 个并发用户    | 针对API调用场景                                              |

### 6.2 可用性与可靠性

| 指标           | 要求                                             |
| :------------- | :----------------------------------------------- |
| **系统可用性** | 99.5%（年停机时间 < 43.8小时）                   |
| **数据持久性** | 99.99%（写入数据库后不丢失）                     |
| **故障恢复**   | 服务崩溃后能在 30秒 内自动重启（借助Docker/K8s） |
| **优雅关闭**   | 收到SIGTERM信号后，完成正在处理的请求再退出      |

### 6.3 安全性需求

| 类别         | 要求                                                         |
| :----------- | :----------------------------------------------------------- |
| **认证**     | 使用API Key（Bearer Token）进行身份认证，所有非`/health`端点均需认证 |
| **授权**     | v1.0暂不实现多租户，所有认证用户拥有相同权限；v2.0考虑资源隔离 |
| **传输安全** | 生产环境必须启用HTTPS                                        |
| **敏感数据** | 不存储任何用户个人信息，仅存储业务指标数据                   |
| **限流**     | 每个API Key每分钟最多 100 次请求（可配置），防止滥用         |
| **输入校验** | 所有用户输入必须通过Pydantic校验，防止SQL注入和恶意数据      |

### 6.4 可扩展性

| 维度           | 要求                                              |
| :------------- | :------------------------------------------------ |
| **水平扩展**   | 应用层无状态，可通过增加容器实例水平扩展          |
| **数据库扩展** | 设计时考虑未来可迁移至TimescaleDB，支持数据分片   |
| **算法扩展**   | 异常检测和预测算法采用策略模式，方便新增算法实现  |
| **API版本化**  | 所有API带版本前缀（如`/api/v1/`），支持多版本共存 |

### 6.5 可维护性

| 维度           | 要求                                                         |
| :------------- | :----------------------------------------------------------- |
| **日志**       | 结构化日志（JSON格式），包含`request_id`用于链路追踪         |
| **监控**       | 暴露Prometheus格式的指标：请求数、延迟分布、错误率、任务队列长度 |
| **代码质量**   | 单元测试覆盖率 > 80%，通过mypy类型检查，通过ruff lint检查    |
| **文档**       | 代码内嵌docstring，API文档自动生成，独立的README包含快速启动指南 |
| **数据库迁移** | 所有schema变更通过Alembic生成迁移脚本，可回滚                |

### 6.6 数据一致性

| 场景           | 要求                                                         |
| :------------- | :----------------------------------------------------------- |
| **数据写入**   | 单条数据点写入必须保证原子性；批量写入允许部分成功（返回失败列表） |
| **缓存一致性** | 写入新数据后，相关的Redis缓存条目失效或更新，确保读到最新数据 |
| **任务状态**   | 预测任务状态变更（PENDING → STARTED → SUCCESS/FAILURE）必须持久化 |

### A. 术语表

| 术语                            | 定义                                                         |
| :------------------------------ | :----------------------------------------------------------- |
| **指标 (Metric)**               | 一个可度量的业务或技术参数，如DAU、CPU使用率                 |
| **数据点 (Datapoint)**          | 一个指标在某个时刻的具体数值，由时间戳和数值组成             |
| **时序数据 (Time Series Data)** | 按时间顺序排列的数据点序列                                   |
| **3σ原则**                      | 又称拉依达准则，假设数据服从正态分布，超出均值±3倍标准差的点为异常 |
| **移动平均 (Moving Average)**   | 对时间序列计算滑动窗口内的平均值，用于平滑短期波动           |
